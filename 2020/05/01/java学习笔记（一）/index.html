1.2计算机基础知识

二进制

1.3 java虚拟机——JVM(Java Virtual Machine)

各平台的翻译 Win版JVM Linux版JVM Mac版JVM

  • JRE(java runtime enviranment):运行环境，包含JVM和运行时所必需的核心类库。
  • JDK(Java development kit):java程序开发包，包含JRE和开发人员使用的工具。
  • 如果想要运行一个已有的Java程序，那么只需安装JRE。
  • 如果想要开发一个 全新的Java程序，那么必须安装JDK。

JDK>>JRE>>JVM

2.2关键字和标识符

  • 关键字：有特殊含义的，被保留的，不允许随意使用的字符
  • 标识符：自定义的名称
    • 字母 数字 $ _ ;不能以数字开头;不能使用关键字
  • 类名规范：首字母大写，后面每个单词的首字母大写（大驼峰式）
  • 方法名/变量名规范：首字母小写，后面每个单词的首字母大写（小驼峰式）

第四章 数据类型转换

4.1自动类型转换（隐式）

数据范围从小到大

long num1=100;
double num2=2.5f;
float num3=30L;

4.2强制类型转换（显式）

范围小的类型 范围小的变量名=（范围小的类型） 原本范围大的数据;

int num1 =(int)100L;

强制类型转化，可能发生数据溢出或精度损失。

int num1=(int)6000000000L;
System.out.println(num1)//1705032704

int num2=(int)3.5
System.out.println(num2)//3

short/byte/char可以数学运算，例如”+”;运算时都会被提升为int类型

char zifu1='A';
system.out.println(zifu1+1);

byte num1=30;
byte num2=40;
//byte result1=num1+num2;报错
int result1=num1+num2;

short num3=50;
short result2=(short)(num1+num3);

boolean类型不能进行数据类型转换

第六章 运算符和表达式

6.1算数运算符

  • | + - * / % ++ — —

++num:先加再用

int num=1;
System.out.println(++num);//2
System.out.println(num);//2

num++:先用再加

int num2=1;
System.out.println(num++);//1
System.out.println(num);//2

只有变量可以使用++ –
30++ 错误！

int x=10,y=20;
result=++x+y--;
System.out.println(result);//31

运算中如果有不同类型的数据，结果是数据范围大的类型。

对于字符产String，加号代表字符串的连接,任何数据类型和字符串连接时，结果都是字符串。

String str1="abc";
System.out.println(str1);//abc
System.out.println("abc"+"def");//abcdef
System.out.println(str1+"def");//abcdef
System.out.println(str1+20);//abc20
System.out.println(str1+20+30);//abc2030
System.out.println(str1+(20+30));//abc50

xx||xx||xx||xx||xx

6.5 三元运算符

数据类型 变量名 =条件判断？表达式A:表达式B

流程：

  1. 判断条件是否成立

  2. 如果成立为true，那么将表达式A的值赋给左侧的变量;如果不成立为false，那么将表达式B的值赋给左边的变量

    int a=10;
    int b=20;
    int max=a>b?a:b;

必须同时保证表达式A和表达式B都符合左侧数据类型的要求

int result =3>4?2.5:10;//报错

三元运算符的结果必须被使用

System.out.println(a>b?a:b);
a>b?a:b;//报错

7.4 编译器的两点优化

对于byte、short、char 如果右边赋值的数值没有超过范围，那么编译器会自动隐含补上（byte）（short）（char）

byte num=/*（byte）*/30;
byte num=128;//报错

char zifu=/*（char）*/65;
System.out.println(zifu)//A

“编译器的常量优化”——在给变量赋值时，如果右边的表达式中全是常量，没有任何变量，那么编译器javac会直接将若干个常量表达式计算得出结果。

short result=5+8;

等号右边全是常量，没有任何变量参与运算，编译之后，得到的.class字节码文件中就是

short result=13;

short num1=10;//正确写法

short a=5,b=8;
short c=a+b//错误写法！左侧需要是int类型

short result=5+8;
System.out.println(zifu)//13

short result2=5+a+8;//错误写法！右边有变量

关系表达式——>true/false

...
}else if(判断条件n){

标准的switch-case

switch(表达式){
    case 常量值1：
        语句体1;
        break;
    case 常量值2：
        语句体2;
        break;
    case 常量值3：
        语句体3;
        break;
    default:
        语句体n+1;
        break;
}

匹配到哪一个case就从哪一个位置向下执行，直到遇到了break或者整体结束为主

穿透的siwtch-case

多个case后面的数值不可以重复

switch后的小括号只能是byte、short、int、char;String、enum

siwtch语句前后顺序可以颠倒，而且break语句可以省略

8.4 三种循环的区别

  • 如果条件判断从没满足过，那么for和while执行0次，do-while执行一次
  • for循环的变量在小括号中定义，只有for循环内部可以使用;
  • while和do-while初始化语句定义在外部，可以继续使用

8.5 条件控制语句 break和continue

switch中直接跳出

循环语句中直接跳出

continue

立刻跳过当前次循环剩余内容，开始下一次循环
