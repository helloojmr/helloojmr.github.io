第一章 继承性

1.1 继承的概念

  • 继承是多态的前提，如果没有继承，就没有多态
  • 继承主要解决的问题就是：共性抽取
  • 父类（基类、超类） 子类（派生类）
  • 继承关系中的特点：
    • 子类可以拥有父类的“内容”
    • 子类还可以拥有自己的内容
    • 内存中：子类对象里面有一个完整的父类对象

1.2 继承的格式

定义父类（一个普通的类定义）

定义子类的格式：

public class 子类名称 extends 父类名称{
 //...
 }

：

//定义一个父类：员工
public class Employee {
    public void method() {
        System.out.println("方法执行");
    }
}

//定义了一个子类：教师
public class Teacher extends Employee {

}

//定义了另一个子类：助教
public class Assistant extends Employee {

}



public class Demo01Extends {

    public static void main(String[] args) {
        //创建了一个讲师子类对象
        Teacher teacher = new Teacher();
        //Teacher类中虽然什么都没写，但会继承来自父类的method方法
        teacher.method();//方法执行

        //创建另一个子类助教的对象
        Assistant assistant = new Assistant();
        assistant.method();//方法执行
    }

}

1.3 继承中成员变量的访问特点

如果子类的成员变量和父类重名，则它有两种访问方式

  1. 直接通过子类对象访问成员变量
  2. 间接通过成员方法访问成员变量

：

public class Demo01ExtendsField {
    public static void main(String[] args) {
        Fu fu = new Fu();//创建父类对象
        System.out.println(fu.numFu);//父类就是普通类

        Zi zi = new Zi();

        System.out.println(zi.numZi);//20
        System.out.println(zi.numFu);//10

        //成员变量重名时：
        System.out.println(zi.num);//200

        zi.methodZi();//200

        //这个方法是属于父类的方法，子类只是拿来继承使用
        zi.methodFu();//100
    }
}
  • 子类方法中三种变量重名情况下的区分
    • 局部变量：直接使用
    • 本类的成员变量：this.变量名
    • 父类的成员变量：super.变量名

:

public class Fu {
        int num = 10;
    }


    public class Zi extends Fu {
        int num=20;
        public void method(){
            int num =30;
            System.out.println(num);//30 局部变量
            System.out.println(this.num);//20 本类的成员变量
            System.out.println(super.num);//10 父类的成员变量
        }
    }

public class Demo01ExtendsField {
    public static void main(String[] args) {
        Zi zi = new Zi();
        zi.method();
    }
}

1.4 继承关系中成员方法的访问特点及方法的覆写（override）

子类的成员变量和成员方法，如果和父类有重名，子类都是自我优先

只有在通过父类成员方法调用重名的成员变量时优先用父类的重名成员变量

  • 重写/覆盖/覆写（override）

概念：在继承关系中，方法的名称一样，参数列表也一样，此时就产生了方法的重写

特点：方法在覆写之后，子类对象调用该名称方法时会优先使用自己的该方法

  • 重写（override）和重载（overload）
  • 重写方法名称一样，参数列表一样
  • 重载方法名称一样，参数列表不一样

:

public class Fu {
    public  void methodFu(){
        System.out.println("父类方法执行！");
    }
    public void method(){
        System.out.println("父类重名方法执行！");
    }
}

public class Zi extends Fu {
    public void methodZi(){
        System.out.println("子类方法执行！");
    }
    public void method(){
        System.out.println("子类重名方法执行！");
    }
}

public class Demo01ExtendsMethod {
    public static void main(String[] args) {
        Zi zi = new Zi();
        zi.methodFu();//父类方法执行！
        zi.methodZi();//子类方法执行！

        zi.method();//子类重名方法执行！
    }
}

1.5 方法覆盖重写的注意事项

  1. 覆写方法的名称相同，参数列表相同。
  2. 子类方法的返回值范围必须小于等于父类方法的返回值范围
  3. 子类方法的修饰符权限必须大于等于父类方法的修饰符权限

@Override:写在覆写方法前面，用来检测是不是有效正确的覆盖重写）

设计原则：对于已经投入使用的类，尽量不要进行修改。定义一个新的类，来重复利用其中共性内容，并且添加改动新内容

//本来的老款手机
public class Phone {
    public void call(){
        System.out.println("打电话");

    }
    public void send(){
        System.out.println("发短信");
    }
    public void show(){
        System.out.println("显示号码");
    }
}

//新款手机
public class NewPhone extends Phone {
    @Override
    public void show() {
        super.show();//把父类的show拿里利用
        System.out.println("显示姓名");
        System.out.println("显示头像");
    }
}

public class Demo01Phone {
    public static void main(String[] args) {
        Phone phone = new Phone();
        phone.call();
        phone.send();
        phone.show();

        NewPhone newPhone = new NewPhone();
        phone.call();
        phone.send();
        phone.show();
    }

}

1.6 父子类构造方法的访问特点

  1. 子类构造方法中，会有一个默认隐含的super()调用,所以一定是先调用的父类构造方法，后执行的子类构造方法
  2. super的父类构造方法调用，必须是子类构造方法的第一条语句，一个子类构造无法调用多次super()
  3. 子类构造方法可以通过super()来调用父类重载构造方法

:

public class Fu {
    public Fu() {
        System.out.println("父类无参构造");
    }

    public Fu(int num) {
        System.out.println("父类有参构造");
    }
}

public class Zi extends Fu {

    public Zi() {
        super(20);//在调用父类重载的构造方法
//        super();//编译器默认调用父类无参构造方法
        System.out.println("子类构造方法");
    }
    public void method(){
//        super();//错误写法！只有子类构造方法，才能调用父类构造方法

    }
}


public class Demo01Constructor {
    public static void main(String[] args) {
        Zi zi = new Zi();
        //父类构造方法
        //子类构造方法
    }
}

1.7 super关键字的三种用法

  • 在子类的构造方法中访问父类的构造方法
  • 在子类的成员方法中访问父类的成员变量
  • 在子类的成员方法中访问父类的成员方法

:

public class Fu {
    int num=10;
    public  void method(){
        System.out.println("父类方法");
    }
}


public class Zi extends Fu {
    int num = 20;

    public void methodZi() {
        System.out.println(super.num);//父类的num

    }

    public void method() {
        super.method();
        System.out.println("子类方法");
    }

    public Zi() {
        super();
    }

}

super关键字用来访问父类内容，this关键字用来访问本类内容
不能同时使用super和this两种构造调用

1.8 this关键字的三种用法

  • 在本类的成员方法中访问本类的成员变量
  • 在本类的成员方法中访问本类的另一个成员方法
  • 在本类的构造方法中访问本类的另一个构造方法,this(…)必须是构造方法的第一个语句

:

    public class Fu {
        int num=30;
    }



    public class Zi extends Fu {
        int num = 20;

        public void showNum() {
            int num = 10;
            System.out.println(num);//局部变量
            System.out.println(this.num);//本来中的成员变量
            System.out.println(super.num);
        }

        public void methodA() {
            System.out.println("AAA");
        }

        public void methodB() {
            this.methodA();
            System.out.println("BBB");
        }

        public Zi() {
                  this(10);//本类的无参构造，调用本类的有参调用
        //        this(1,2);//错误写法！
        //        super();//不能同时使用super和this两种构造调用
        }

        public Zi(int n) {
            this(20, 30);

        }

        public Zi(int n, int m) {
            //默认调用super()
        }
}
