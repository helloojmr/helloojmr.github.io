第一章 Collection集合

1.1 集合概述

  • 集合：集合是java中提供的一种容器，可以用来存储多个数据。
  • 数组的长度是固定的。集合的长度是可变的。
  • 数组中存储的是同一类型的元素，可以存储基本数据类型值。集合存储的都是对象,而且对象的类型可以不一致,在开发中一般对象多的时候，使用集合进行存储。
  • 集合按照其存储结构可以分为两大类，分别是单列集合java.util.Collection和双列集合java.util.Map
  • Collection：单列集合类的根接口，用于存储一系列符合某种规则的元素。有两个重要的子接口，分别是java.util.List和java.util.Set。
  • List的特点是元素有序、可重复。其主要实现类有java.util.ArrayList、java.util.LinkedList、java.util.Vector;
  • Set的特点是元素无序、不可重复。其主要实现类有java.util.HashSet和java.util.TreeSet、LinkedHashSet(有序，HashSet子类)。

1.2 Collection 常用功能

Collection是所有单列集合的父接口，因此在Collection中定义了单列集合(List和Set)通用的一些方法，这些方法可用于操作所有的单列集合。方法如下：

  • public boolean add(E e)： 把给定的对象添加到当前集合中 。
  • public void clear() :清空集合中所有的元素。
  • public boolean remove(E e): 把给定的对象在当前集合中删除。
  • public boolean contains(E e): 判断当前集合中是否包含给定的对象。
  • public boolean isEmpty(): 判断当前集合是否为空。
  • public int size(): 返回集合中元素的个数。
  • public Object[] toArray(): 把集合中的元素，存储到数组中。

：

import java.util.ArrayList;
import java.util.Collection;

public class Demo01Collection {
    public static void main(String[] args) {


        Collection<String> coll = new ArrayList<>();
        System.out.println(coll);//[]
        boolean b1 = coll.add("WUWUW");
        System.out.println(b1);//true
        System.out.println(coll);//[WUWUW]
        coll.add("HINI");
        System.out.println(coll);//[WUWUW, HINI]


        boolean b2 = coll.remove("WUWUW");
        System.out.println(b2);//true
        System.out.println(coll);//[HINI]
        boolean b3 = coll.remove("HA");
        System.out.println(b3);//false
        System.out.println(coll);//[HINI]


        boolean b4 = coll.contains("HINI");
        System.out.println(b4);//true
        boolean b5 = coll.contains("FD");
        System.out.println(b5);//false


        boolean b6 = coll.isEmpty();
        System.out.println(b6);//false


        int b7 = coll.size();
        System.out.println(b7);//1


        Object[] arr = coll.toArray();
        for (int i = 0; i < arr.length; i++) {
            System.out.println(arr[i]);//HINI
        }

        coll.clear();
        System.out.println(coll);//[]
        System.out.println(coll.isEmpty());//true


    }
}

第二章 Iterator迭代器

2.1 Iterator接口

在程序开发中，经常需要遍历集合中的所有元素。针对这种需求，JDK专门提供了一个接口java.util.Iterator。Iterator接口也是Java集合中的一员，但它与Collection、Map接口有所不同，Collection接口与Map接口主要用于存储元素，而Iterator主要用于迭代访问（即遍历）Collection中的元素，因此Iterator对象也被称为迭代器。

  • 迭代：即Collection集合元素的通用获取方式。在取元素之前先要判断集合中有没有元素，如果有，就把这个元素取出来，继续再判断，如果还有就再取出来。一直到把集合中的所有元素全部取出。
  • 迭代：重复、改进、认知升级

Iterator接口的常用方法如下：

  • public E next():返回迭代的下一个元素。

  • public boolean hasNext():如果仍有元素可以迭代，则返回true。

  • Iterator迭代器是一个接口，无法直接使用，需要使用Iterator接口的实现类对象，Collection接口有一个方法，叫iterator(),这个方法可以返回一个迭代器的实现类对象,使用Iterator接口接收（多态）

  • Iterator 接口是有泛型的，迭代器的泛型和集合一样，集合是什么泛型，迭代器就是什么泛型

2.2 增强for

增强for循环(for each循环)是JDK1.5以后出来的一个高级for循环，专门用来遍历数组和集合的。它的内部原理其实是个Iterator迭代器，所以在遍历的过程中，不能对集合中的元素进行增删操作。

for(元素的数据类型  变量名 : Collection集合/数组){
      //...
}

:

import java.util.ArrayList;
import java.util.Collection;

public class Demo01For {
    public static void main(String[] args) {

        Collection<Integer> coll= new ArrayList();
        coll.add(1);
        coll.add(2);
        coll.add(3);
        coll.add(4);
        for(Integer i : coll){
            System.out.println(i);
        }

        int[] s={1,2,3,4};
        for(int i:s){
//          System.out.println(s[i]);ArrayIndexOutOfBoundsException
            System.out.println(i);
        }


    }


}

新for循环必须有被遍历的目标。目标只能是Collection或者是数组。新式for仅仅作为遍历操作出现。

第三章 泛型

3.1 泛型的概述

  • 泛型：可以在类或方法中预支地使用未知的类型。
  • 一般在创建对象时，将未知的类型确定具体的类型。当没有指定泛型时，默认类型为Object类型。

:

import java.util.ArrayList;
import java.util.Iterator;

public class Demo01Generic {

/*将运行时期的ClassCastException转移为编译时期编译失败,避免了类型强转的麻烦*/
    public static void main(String[] args) {
        demo1();
        demo2();
    }

    private static void demo1() {
        ArrayList list = new ArrayList();
        list.add(1);
        list.add('Q');
        list.add("A");
        Iterator iterator = list.iterator();
        while (iterator.hasNext()) {
            //遍历的元素是Object类型
            Object obj = iterator.next();
            System.out.println(obj);

//            想要使用String类特有的length()方法获取字符串的长度
//            向下转型
//            String S = (String) (obj);//ClassCastException：Integer cannot be cast to class String
//            System.out.println(S.length());
        }

    }

    public static void demo2() {
        ArrayList<String> list = new ArrayList<>();
        list.add("123");
        list.add("ABC");
        Iterator<String> iterator = list.iterator();
        while (iterator.hasNext()) {

            String S = iterator.next();
            System.out.println(S.length());
        }


    }
}

使用泛型：在创建对象的时候确定泛型

:

public class GenericClass<E> {
    private String name;
    private E unKnow;

    public E getUnKnow() {
        return unKnow;
    }

    public void setUnKnow(E unKnow) {
        this.unKnow = unKnow;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}


public class Demo02GenericClass {
    public static void main(String[] args) {
        //创建对象时不写泛型默认为Object类型
        GenericClass gc = new GenericClass();
        Object unKnow1 = gc.getUnKnow();

        //泛型使用Integer类型
        GenericClass<Integer> gc2 = new GenericClass<>();
        gc2.setUnKnow(1);
        Integer integer = gc2.getUnKnow();
        System.out.println(integer);//1

        //泛型使用String类型
        GenericClass<String> gc3 = new GenericClass<>();
        gc3.setUnKnow("WU");
        String string = gc3.getUnKnow();
        System.out.println(string);//WU


    }
}

public class GenericMethod {

public class DemoGeneric {

3.3 泛型通配符

当使用泛型类或者接口时，传递的数据中，泛型类型不确定，可以通过通配符<?>表示。但是一旦使用泛型的通配符后，只能使用Object类中的共性方法，集合中元素自身方法无法使用。

泛型的通配符:不知道使用什么类型来接收的时候,此时可以使用<?>表示未知通配符。
